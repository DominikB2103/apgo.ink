How to grow

Add a product only when the method is ready.

01

Name the public need

Start with the decision a reader cannot currently make well—not with a fashionable format.

02

Publish the rules

Define inputs, exclusions, update frequency, correction path and who remains accountable.

03

Launch small

Ship a limited pilot, invite criticism and keep claims proportional to the evidence.

04

Archive change

Preserve old versions so improvement never becomes quiet revisionism.

Build in public

Fund the next useful thing.

Support goes to research, data, design and engineering. It never buys a result.
